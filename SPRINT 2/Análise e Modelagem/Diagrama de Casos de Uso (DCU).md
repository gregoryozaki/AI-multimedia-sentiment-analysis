## Diagrama de Casos de Uso no EMODIA ##

O Diagrama de Casos de Uso (DCU) é uma ferramenta fundamental da Linguagem de Modelagem Unificada (UML) utilizada para descrever a funcionalidade do sistema do ponto de vista do usuário. 
Ele é um diagrama comportamental que define o escopo do EMODIA, mapeando o que o sistema fará para atender às necessidades dos seus atores (usuários).

## Para o projeto EMODIA, o DCU é crucial para: ## 

Identificar Atores: Claramente distingue os usuários primários que interagem com a plataforma, como o Usuário Cadastrado e o Especialista/Psicólogo.
Definir Funções: Representa as principais funcionalidades do sistema (os "casos de uso") que correspondem diretamente às suas Histórias de Usuário (US01 a US25), como Registrar Emoções, Gerar Relatórios e Receber Alerta de Risco.
Estabelecer Limites: O diagrama delimita o escopo do EMODIA, mostrando onde ele interage com sistemas externos, como a Plataforma de Análise IA e os Serviços de Emergência, sem modelar a lógica interna desses sistemas.
Portanto, o Diagrama de Casos de Uso serve como um contrato visual entre o cliente e a equipe de desenvolvimento, assegurando que o sistema EMODIA entregue precisamente as funcionalidades essenciais para o monitoramento e auxílio emocional.
